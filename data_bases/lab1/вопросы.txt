1. DDL, DML, DCL
	DDL: CREATE, ALTER, DROP, TRUNCATE - используется для определения таблиц, представлений, типов
	DML: SELECT, INSERT INTO, DELETE, UPDATE - для получения и изменения записей в таблицах
	DCL: GRANT, REVOKE - права доступа к таблицам
	TCL (transaction control l): BEGIN, COMMIT, ROLLBACK - для работы с транзакциями
2. DROP DELETE TRUNCATE
	DELETE: удаляет конкретные записи, с условиями, работает с внешними ключами
	TRUNCATE: полностью очищает таблицу, не работает с внешними ключами, DDL, нельзя откатить, работает быстро
	DROP: полностью удаляет таблицу (или еще что), все ключи, связи и тд

4. ON DELETE/UPDATE
	CASCADE - удаляет объекты, зависящие от удаляемого
	RESTRICT - не позволяет удалить, если есть зависимые
	SET DEFAULT/NULL - устанавливают дефолтное или нал
	NO ACTION - при существовании связанных записей создает ошибку, но в конце транзакции
